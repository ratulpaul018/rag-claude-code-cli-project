# DOC_QA_AG_RAG - Agentic RAG Document Analysis

Standalone Agentic RAG system using LangGraph for intelligent multi-step document analysis.

## What is Agentic RAG?

Unlike basic RAG which does simple retrieval + generation, **Agentic RAG** uses a three-stage intelligent workflow:

1. **Retrieval Agent** - Analyzes the question and formulates optimal search queries
2. **Analysis Agent** - Examines retrieved documents and extracts key information
3. **Answer Agent** - Synthesizes findings into comprehensive answers with reasoning steps

This results in **deeper analysis** and **better answers** for complex questions.

## Features

✅ **Intelligent Multi-Step Reasoning** - LangGraph StateGraph orchestration
✅ **Document-Aware Analysis** - Understands context across chunks
✅ **Fast PDF Processing** - PyMuPDFLoader for reliable extraction
✅ **Ollama-Generated Suggestions** - AI-powered recommended questions after upload
✅ **Interactive Web UI** - Clean interface with chat and source references
✅ **Source Attribution** - Click references to view PDF pages
✅ **Vector Search** - ChromaDB with semantic understanding
✅ **Local Processing** - Private, no external APIs
✅ **Auto-Loading** - Vector store automatically loads on Flask startup
✅ **Professional Follow-up** - Smart follow-up message after each answer

## Installation

```bash
# Install dependencies (from parent directory)
pip install -r ../requirements.txt

# Make sure Ollama is running
ollama serve

# Run the Agentic RAG app (in another terminal)
python web_app_up.py
```

Then open **http://127.0.0.1:7000** in your browser.

2. Click "Read The PDF" to process

### Ask Questions
1. Type your question in the input field
2. System analyzes and reasons through the answer

- **Reset** - Clear all documents and start fresh

## Configuration

Edit `agentic_rag_doc_analysis.py` to customize:

```python
OLLAMA_MODEL = "llama3.2:latest"       # LLM model
EMBEDDING_MODEL = "nomic-embed-text"   # Embedding model
CHUNK_SIZE = 900                        # Characters per chunk
CHUNK_OVERLAP = 250                     # Chunk overlap
RETRIEVAL_K = 15                        # Context chunks to retrieve
```

## API Endpoints

### Upload PDF
```bash
POST /api/upload
Content-Type: multipart/form-data

Response: {
  "success": true,
  "chunks": 150,
  "filename": "document.pdf",
  "mode": "agentic",
  "suggested_questions": [
    "What is the main topic?",
    "What are the key points?",
    "..."
  ]
}
```
**Note:** Suggested questions are AI-generated by Ollama from document chunks.

### Ask Question
```bash
POST /api/ask
Content-Type: application/json

Body: { "question": "Your question here" }

Response: {
  "success": true,
  "answer": "Detailed answer with reasoning...",
  "sources": [{"page": 5, "content": "..."}],
  "mode": "agentic",
  "follow_up": "❓ What more can I help you with?"
}
```
**Note:** Follow-up message encourages continued interaction after each answer.

### Get Status
```bash
GET /api/status

Response: {
  "vector_store_loaded": true,
  "qa_chain_ready": true
}
```

### Reset System
```bash
POST /api/reset

Response: {
  "success": true,
  "message": "System reset..."
}
```

## Key Improvements

### Auto-Loading Vector Store
When Flask starts, it automatically loads any existing vector store from disk. This means:
- Users don't need to re-upload PDFs after Flask restarts
- Vector store persists across sessions
- Immediate availability for Q&A without waiting

### Ollama-Generated Questions
Suggested questions are generated intelligently by Ollama:
- Based on document chunks
- Specific and answerable from the document
- Professional and clear
- Helps users explore document content

### Professional Follow-up
After each answer, a friendly follow-up message encourages users to ask more questions, improving engagement.

## Troubleshooting

### Ollama Not Running
```bash
# Make sure Ollama is running in another terminal
ollama serve

# Then run Flask in a new terminal
python web_app_up.py
```

### Vector Store Not Loading on Startup
- Check if `chroma_db/` directory exists and has data
- Ensure `chroma_db/chroma.sqlite3` file exists
- Check Flask startup logs for errors

### Out of Memory
- Reduce RETRIEVAL_K (from 15 to 5-10)
- Reduce CHUNK_SIZE (from 900 to 500)

### Slow Responses
- This is normal for Agentic RAG
- Complex questions take 8-15 seconds
- Each request goes through 3 reasoning stages

### Questions Not Generating
- Check that document chunks were created (chunks > 0)
- Verify Ollama is running on port 11434
- Check Flask logs for Ollama errors
